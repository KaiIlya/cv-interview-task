# Задание для собеседования

В этом репозитории лежит намеренно проблемный прототип решения задачи классификации изображений.

Ваша задача - превратить notebook в чистый, корректный и поддерживаемый training pipeline.

Ожидаемое время выполнения: 40 минут.

## Что нужно сделать

1. Настроить окружение.
2. Скачать CIFAR-10 через код базового решения.
3. Запустить notebook с базовым решением.
4. Изучить пайплайн данных, модель, цикл обучения, валидацию, метрики, логирование, checkpoints и графики.
5. Найти проблемы, из-за которых модель не обучается корректно или метрики недостоверны.
6. Отрефакторить код в понятную проектную структуру.
7. Запустить обучение после исправлений и показать результат на валидации.
8. Объяснить, что было изменено и почему.

Можно менять любой код, создавать новые модули, скрипты, конфиги, тесты или notebook-файлы.

## Установка и запуск

```bash
git clone <repo-url>
cd cv-interview-task

python -m venv .venv
source .venv/bin/activate

pip install -r requirements.txt
jupyter notebook notebooks/01_dirty_baseline.ipynb
```

Для Windows PowerShell:

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1

pip install -r requirements.txt
jupyter notebook notebooks/01_dirty_baseline.ipynb
```

Также можно быстро запустить базовое решение без notebook:

```bash
python scripts/run_dirty_baseline.py
```

После запуска baseline создает артефакты:

```text
runs/dirty_baseline/
  config.json
checkpoints/
  last.pt
  best.pt
```

TensorBoard можно открыть командой:

```bash
tensorboard --logdir runs
```

## Датасет

CIFAR-10 скачивается через `torchvision` при первом запуске. Baseline использует только небольшой subset, чтобы собеседование не превращалось в ожидание обучения:

- обучающая подвыборка: 1 500 изображений;
- валидационная подвыборка: 500 изображений;
- классы: 10 классов CIFAR-10.

Задача намеренно небольшая. Цель не в достижении state-of-the-art качества.

## Что ожидается в результате

К концу задания нужно предоставить:

- краткое описание найденных проблем;
- отрефакторенную структуру проекта;
- рабочую команду или notebook для обучения;
- метрики на валидации после исправлений;
- графики или примеры предсказаний модели;
- сохраненный checkpoint модели;
- понятное логирование эксперимента;
- краткое объяснение архитектурных решений.

## Примечания

Baseline специально написан как плохой исследовательский прототип. Относитесь к нему как к реальному notebook, который достался от другой команды: сначала разберитесь, проверьте предположения, найдите критичные ошибки, затем улучшайте структуру.
